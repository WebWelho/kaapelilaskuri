## Quality Rules

Rules enforced during the post-execution review. Add, remove, or edit these.

- No `: any` in TypeScript (use `unknown` + type narrowing)
- No `console.log` in production code
- No hardcoded colors — use CSS variables or design tokens
- No `useEffect` for data fetching — use React Query or SWR
- All DB query results must be null-checked before access
- No secrets or API keys committed to source control

## Anti-patterns to Watch (pre-check additions)

Add project-specific patterns you have burned yourself with before.
The engine will check for these before starting any implementation.

```
Pattern: [name]
Detection: [grep-able string or heuristic]
Why: [one sentence]
Correct approach: [one sentence]
```

Example:
```
Pattern: Direct Supabase query from component
Detection: `supabase.from(` inside a React component file
Why: Bypasses the repository layer and couples UI to DB schema
Correct approach: Use a repository function from /lib/repositories/
```
